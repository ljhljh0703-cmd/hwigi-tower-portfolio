소형 LLM으로
게임 NPC 만들기

2.4B 파라미터 한국어 모델로 캐릭터 일관성을 가진 AI NPC를 만든 과정. 실패한 모델 선택, 토큰화 버그, 언어 누출 문제를 거쳐 마타이오스 대화 파이프라인을 구성했습니다.

이 페이지는 전투 RL 사례가 아니라 NPC 대화 모델 파인튜닝 사례입니다. 본편 전투 동행자 행동은 deterministic rule로 처리하고, 여기서는 캐릭터 대화와 세계관 응답을 위한 소형 LLM 실험을 다룹니다.

프로젝트 배경

회귀자는 탑을 오른다는 모바일 세로형 로그라이크 프로토타입입니다. 마타이오스는 단순 조력자 UI가 아니라, 전투와 선택, 기억을 통해 플레이어 옆에 있는 감각을 만들어야 하는 핵심 동행자입니다.

소형 LLM이 어려웠던 이유

일반 API를 붙이면 빠르게 데모는 만들 수 있습니다. 하지만 인디 게임의 비용, 오프라인 가능성, 대화 레이턴시, 캐릭터 제어를 동시에 고려하면 작은 로컬 모델을 실험할 이유가 있었습니다.

P1

비용과 레이턴시

대형 API는 대화당 비용과 1-3초 레이턴시가 있고, 오프라인 동작도 어렵습니다. 반복 플레이 게임의 동행자 대화에는 부담이 큽니다.

P2

한국어 일관성

Qwen2.5 같은 소형 다국어 모델은 학습 데이터가 부족하면 영어/중국어로 전환되는 문제가 발생했습니다.

P3

캐릭터 제어

호감도, 스토리 단계, BIW 차단, 기억 반응을 동시에 제어해야 했습니다. 단순 챗봇이 아니라 게임 캐릭터여야 했습니다.

실패 기록

실패 기록을 숨기지 않았습니다. 모델 호환성, 다국어 prior, 데이터 증강 한계, BPE 토큰화 버그가 실제 병목이었습니다.

실패 01

HCX-SEED 0.5B: 한국어 특화 기대, GGUF/응답 제어 실패

한국어 모델이라 기대했지만 GGUF 변환과 아키텍처 호환 문제가 있었고, BIW 질문에 실제 날씨 정보를 생성하는 등 캐릭터/세계관 제어도 실패했습니다.

실패 02

Qwen2.5-1.5B v6.1: 44% 영어/중국어 누출

ChatML과 GGUF 지원은 좋았지만 9건 테스트 중 4건에서 영어 또는 중국어 응답이 나왔습니다. 622건 SFT로는 다국어 prior를 덮기 어려웠습니다.

실패 03

Qwen2.5 v6.2: BIW 데이터 증강으로도 개선 없음

BIW 거부 데이터와 한국어 강제 프롬프트를 추가했지만 언어 누출이 유지되었습니다. 데이터 추가로 해결되지 않는 구조적 문제라고 판단했습니다.

버그

DataCollator 사일런트 실패: 학습 토큰 0개

response_template 끝 공백이 BPE 문맥 토큰화와 충돌해 응답 경계를 찾지 못했습니다. 에러 없이 labels가 모두 -100이 되어 학습이 사실상 비어 있었습니다.

독립 토큰화: "마타이오스: " -> [..., 25, 220] 문맥 토큰화: "마타이오스: ...좋진 않아." -> [..., 25, 4593] 결과: 응답 경계 탐지 실패 -> labels all -100 -> learning tokens 0 해결: trailing space 제거 + token id list 전달 + label validation 셀 추가

해법: EXAONE 3.5 2.4B

해결은 더 큰 모델을 쓰는 것이 아니라, 문제에 맞는 prior를 가진 모델을 고르는 것이었습니다. 한국어가 강한 EXAONE 3.5 2.4B로 전환하고 QLoRA 설정과 데이터 구성을 함께 조정했습니다.

모델

EXAONE 3.5 2.4B-Instruct

한국어 응답 안정성이 모델 선택의 1순위였습니다. Qwen 계열에서 반복된 다국어 누출 리스크를 줄이기 위해 한국어 prior를 우선했습니다.

주장 경계

무엇을 했고, 무엇을 하지 않았나

이 사례는 NPC 대화 모델 파인튜닝과 Unity 연동 파이프라인입니다. 모바일 온디바이스 EXAONE 배포나 본편 runtime RL/ONNX 연결을 완료했다고 주장하지 않습니다. 전투용 마타이오스 brain은 별도 deterministic rule로 구현되어 있습니다.

"AI NPC를 만든다는 것은 모델 하나를 붙이는 일이 아니라, 캐릭터 계약, 실패 테스트, 데이터 보강, 런타임 경계를 함께 설계하는 일이다."
